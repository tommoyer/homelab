## Caddy Reverse Proxy (LXC)

This folder is intentionally minimal.

- Caddy runs in an LXC container and is managed by `systemd` (installed via a Proxmox helper script).
- `proxy/Caddyfile` is generated by other tooling in this repo; treat it as an output artifact.

### What lives where (in the LXC)

- Caddyfile: `/etc/caddy/Caddyfile`
- Caddy data/config: typically `/var/lib/caddy` and `/var/log/caddy` (package dependent)
- Environment file (for DNS providers / email): package dependent; verify with:
	- `systemctl cat caddy | sed -n '/EnvironmentFile/p'`

If your generated Caddyfile uses `{env.CLOUDFLARE_API_TOKEN}` (DNS-01), you must ensure that env var is available to the Caddy service.

### Configure secrets (Cloudflare token, ACME email)

1. Determine the `EnvironmentFile=` path from the service definition.
2. Create that file on the container from [proxy/caddy.env.example](caddy.env.example).
3. Lock it down (recommended): `chmod 600 <envfile>`.

### Deploy / reload

If you’re using the generator’s deploy feature, it should scp the generated Caddyfile to the container and run a restart/reload command.

For a manual workflow from your workstation:

```bash
cd proxy
task deploy PROXY_HOST=root@proxy.dmz.moyer.wtf
```

This copies [proxy/Caddyfile](Caddyfile) to `/etc/caddy/Caddyfile`, validates it, then runs `systemctl reload caddy`.

### Notes on Cloudflare DNS plugin

The stock distro Caddy package may not include the `caddy-dns/cloudflare` module.
If Caddy fails to load `tls { dns cloudflare ... }`, you need a Caddy build that includes that module.

If you installed Caddy via the Proxmox VE Helper Script, its docs note:

- Internal modules: `caddy add-package PACKAGENAME`
- External modules (like Cloudflare DNS): `xcaddy build --with github.com/caddy-dns/cloudflare`

This repo includes a small bootstrap script that automates the `xcaddy` path for Cloudflare DNS and installs the resulting binary, then restarts the service:

- Run inside the LXC: [proxy/bootstrap-caddy-cloudflare.sh](bootstrap-caddy-cloudflare.sh)
- Run from your workstation: `cd proxy && task install-cloudflare-module PROXY_HOST=root@proxy.dmz.moyer.wtf`

Note: `xcaddy` may need manual updates after a Caddy update.
